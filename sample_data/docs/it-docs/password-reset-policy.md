# IT Policy: Password Reset Verification

Document type: IT documentation

## Purpose

Help desk analysts must verify user identity before initiating a password reset or MFA reset.

## Requirements

Analysts must confirm the requester through approved verification channels. Do not ask users to send passwords, one-time passcodes, recovery codes, or personal secrets in a ticket. If a ticket includes a password or token, advise the user to rotate it and remove the secret according to policy.

## Approved Response

Tell the user that support can help reset access after verification. Provide the approved verification workflow and avoid collecting sensitive authentication material in chat or ticket comments.
